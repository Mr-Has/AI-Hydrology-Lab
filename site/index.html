Exercise 1

Finding very low river flows

Start with a river-flow record and identify unusually low water conditions. The hydrology idea is a percentile: Q90 or Q95 describes a flow level that most days are above.

Exercise 2

Explaining drought with rainfall

A rainfall record is converted into a drought score. SPI compares recent rainfall with what is normal for that place, where negative values mean drier conditions.

Exercise 3

Checking whether a model is doing well

Measured streamflow is compared with model predictions. NSE and KGE are scores that show whether the model follows the observed river behavior closely enough to trust.

Exercise 5

Reusing good hydrology habits

Repeated hydrology checks are turned into reusable instructions. The goal is to avoid common flood-analysis mistakes, such as mixing up water years, missing values, or rank conventions.

Exercise 6

Estimating rare floods

Annual peak flows are used to estimate events such as a 10-year or 100-year flood. The LP3 model fits a probability pattern to past flood peaks so rare flood sizes can be estimated.

View notebook on GitHub

Exercise 7

Getting water-model data safely

The National Water Model contains many files. The hydrology idea is to narrow the data request before taking action, so the work stays small, traceable, and focused on the right location or time period.

View notebook on GitHub

Exercise 8

Reading stream-gauge data correctly

USGS-style streamflow data describe measurements through time. Each value only makes sense when its date, unit, site, and field name are read correctly.

View notebook on GitHub

Exercise 9

Comparing drought over different time windows

The same drought calculation is run over short and longer rainfall windows. A 1-month SPI can show quick dry spells, while 3- or 6-month SPI values show slower seasonal drought patterns.
