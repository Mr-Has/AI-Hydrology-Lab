Plain-language guide

What each exercise is about

This guide explains the goal of each notebook before it is opened. The language is kept simple so the workflow and model ideas are clear without assuming a programming background.

01

Finding very low river flows

Start with a river-flow record, make a short plan, then use code to identify unusually low water conditions. The model idea is a percentile: Q90 or Q95 describes a flow level that most days are above.

02

Explaining drought with rainfall

A rainfall record is converted into a drought score. The SPI idea is simple: compare recent rainfall with what is normal for that place, where negative values mean drier conditions.

03

Checking whether a model is doing well

Measured streamflow is compared with model predictions. NSE and KGE are diagnostic scores: they show whether the model follows the observed river behavior closely enough to trust.

04

Keeping project instructions consistent

A project needs stable rules, such as units, time step, and data checks. This workflow exercise helps every coding session start with the same scientific expectations.

05

Reusing good hydrology habits

Repeated hydrology checks are turned into reusable instructions. The workflow is meant to avoid common flood-analysis mistakes, such as mixing up water years, missing values, or rank conventions.

06

Estimating rare floods

Annual peak flows are used to estimate events such as a 10-year or 100-year flood. The LP3 model fits a probability pattern to past flood peaks so rare flood sizes can be estimated.

07

Getting water-model data safely

The exercise plans a data request before downloading anything. The order is to check the command, list what is available, choose a small target, preview it, and only then download.

08

Reading stream-gauge data correctly

A USGS-style streamflow response is used as the example. The data idea is a time series: a sequence of flow measurements over time that must be read with the correct field names, units, and dates.

09

Comparing drought over different time windows

The same drought calculation is run over short and longer rainfall windows. A 1-month SPI can show quick dry spells, while 3- or 6-month SPI values show slower seasonal drought patterns.
